# 管理中心与设置页 UI 整理

## 功能概述

v0.7.0 将标签、工具和 Skills 自动更新统一收拢到管理中心；设置页回归应用级偏好配置。这样可以避免顶部导航和设置页继续承载过多业务入口，同时让用户在一个固定位置完成常用管理操作。

## 背景与目标

- 标签、工具和自动更新都属于高频管理能力，分散在多个入口会增加查找成本。
- 设置页需要聚焦语言、外观、存储、GitHub 访问、网络代理和应用版本更新，避免和业务管理混在一起。
- 管理中心需要和首页、探索页保持一致的页面边距和视觉密度，减少页面之间的跳变感。
- 设置页内容较多，需要按任务分组展示，让用户快速理解每组设置的作用。

## 主要变更

### 管理中心布局

- 主导航保留「管理中心」入口，内部通过「标签 / 工具 / 更新」切换不同管理页面。
- 管理中心使用与主工作区一致的左右边距：桌面端保持紧凑工作区，移动端自动收窄。
- 管理中心页头右侧只保留页面切换，不再展示每个子页面的状态统计，避免右上区域内容随 tab 变化而显得拥挤。
- 标签、工具和更新页面继承统一布局，减少切换时的视觉跳动。

### 设置页重构

- 返回入口并入标题行，不再单独占用一整行。
- 顶部展示设置标题、说明，以及界面语言、外观模式、应用版本三个摘要项。
- 设置项按任务分为四组：
  - 界面：语言和外观模式。
  - 存储与缓存：Skills 存储路径、缓存保留时间、重复拉取间隔。
  - GitHub 与网络：GitHub Token、网络代理和代理端口。
  - 应用更新：应用版本与检查更新。
- 设置页不再承载标签、工具和 Skills 自动更新入口。

### 文案整理

- 「Git 缓存清理（天）」调整为「缓存保留时间（天）」，强调它控制缓存文件保留多久。
- 「Git 缓存新鲜期（秒）」调整为「重复拉取间隔（秒）」，强调它用于避免短时间内重复拉取。
- 中文和英文资源同步更新，确保设置项含义一致。

### 更新页整理

- 管理中心的「更新」页展示系统定时更新开关、更新计划和手动立即更新入口。
- 上次运行结果在页面内展示检查数量、更新数量、失败数量、开始时间、完成时间和耗时。
- 失败项通过页面内「需要处理」区域展示，保留复制详情能力，不再使用独立进度弹窗承载主要信息。
